Save Files

All save game files are found in the SAVES directory, and numbered as SAVE00.DAT - SAVE04.DAT. A single save file contains all the information about the players current state, such as HP, mana, stats, inventory, etc. Note that the save file stores values in Little Endian format. The offsets for each value are as follows:

Inventory:

I have found inventory editing to be quite difficult, and this program does not yet support being able to edit or view inventory. However, I have made some discoveries on the issue, and will write here what I know. Hopefully it will help someone figure it out!

Anvil of Dawn seems to track inventory by in which order it was placed in the inventory. For example, if you have two items "A" and "B", if you place "A" in your inventory first, then item "A" will be the item in [slot 1]. If you were to place "B" in your inventory first, then "B" would be the item in [slot 1]. If you have multiple items in your inventory and remove one, then the gap gets "filled" by all item slots that appear after it shifting by 1. It is not known to me how storage items (such as sacks) function within this data structure.

From what I could tell, each "item slot" is 23 bytes apart, starting at offset 0x4E25 (slot 1). It should be noted that simply writing in values 23 bytes after your last item to "add" a free item to your inventory doesn't usually work. There are other values involved that are unknown to me. I have only managed to edit existing items, not add new ones. Here is an inventory slots example:

ValueOffsetLength
Slot 1 item [X position]0x4E254
Slot 1 item [Y position]0x4E294
Slot 1 Item Sprite ID0x4E2D4
Slot 1 Item Actual ID0x4E312
+23 bytes....
Slot 2 item [X position]0x4E3C4
Slot 2 item [Y position]0x4E404
Slot 2 Item Sprite ID0x4E444
Slot 2 Item Actual ID0x4E482
+23 bytes....etc.

Take careful note that the "Sprite ID" and "Actual ID" after two different things. The Sprite ID is just what sprite the engine loads while you're viewing your inventory. The "Actual ID" is what sprite will appear when you grab the item with your cursor, and what functionality it has. The Sprite ID takes the following form (hex):

xx 01 06 80

Where "xx" is any value between 00 and FF. Once FF is reached, it ticks back over to "00" and "01" is incremented to "02". Example:

FF 01 06 80 ---> 00 02 06 80

Music Files

All music files in Anvil of Dawn can be found in RES.008 in the DATA folder. The music is stored as HMP files. HMP is somewhat of an obscure format similar to regular MIDI, and unfortunately there isn’t too much documentation online for it. Luckily, Markus Hein at this website has made a utility for both ripping HMP files, and converting HMP -> MIDI.

Since the HMP files are stored sequentially (that is, one appears right after another ends) the byte length of each HMP file is the difference between the next offset and the current offset you want to find the length for. For example, the byte length of Track 00 is Track 01’s offset (0x7D4C) subtract Track 00’s offset (0x118) so the byte length is 31796(decimal). The offsets for the HMP files are as follows:
